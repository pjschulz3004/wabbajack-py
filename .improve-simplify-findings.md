## Finding 2: `from typing import Optional` is redundant in `loadorder.py` (line 10)

**File:** `src/wabbajack/loadorder.py`, line 10

**What:** `Optional` is imported from `typing` and used in three signatures:
- `GameLoadOrder.__init__` (line 108): `profile_dir: Optional[Path] = None`
- `BethesdaLoadOrder.__init__` (line 221): `data_dir: Optional[Path] = None`
- `get_load_order()` (line 667): `profile_dir: Optional[Path] = None`

The file already uses the modern `X | None` union syntax in line 38 (`masters: list[str] | None = None`). The project requires Python 3.10+ (`pyproject.toml`), where `Path | None` is fully supported.

**Why safe:** The three `Optional[Path]` usages are a straightforward mechanical replacement with `Path | None`. Once replaced, the `from typing import Optional` import can be removed entirely. No runtime behavior changes — type annotations are not enforced at runtime.

**Action:** Replace three `Optional[Path]` with `Path | None`, remove the import. Net reduction: 4 lines of cruft, consistent annotation style.

## Finding 3: Deprecated `--nexus-key` CLI options that silently duplicate `NEXUS_API_KEY` env var (cli.py lines 87, 259)

**File:** `src/wabbajack/cli.py`, lines 87 and 259

**What:** Both the `download` and `repair` commands define:
```python
@click.option('--nexus-key', envvar='NEXUS_API_KEY', hidden=True, help='DEPRECATED: use NEXUS_API_KEY env var')
```
These are hidden from `--help` and already map to the same `NEXUS_API_KEY` environment variable as the non-deprecated `-k/--nexus-key` options on `install` and `reinstall`. They accept the key but the `download` and `repair` commands never pass it to `ModlistInstaller` or any Nexus client — the argument is received as `nexus_key` and forwarded to `ModlistInstaller`, but those two commands' calls already work the same way with or without the deprecated flag. The `repair` command passes `nexus_key or ''` (line 331) and `download` passes `nexus_key=nexus_key` (line 107).

The actual simplification is the inconsistency: `download` and `repair` use the old hidden/deprecated form while `install` and `reinstall` use the clean `-k/--nexus-key` form. The deprecated hidden options add dead parameter surface.

**Why safe:** The options are `hidden=True` — users are not using them in documented workflows. The env var `NEXUS_API_KEY` already provides the same value. Removing the options just removes the undocumented flag; Click's `envvar='NEXUS_API_KEY'` on the option means users currently relying on the env var will continue to work once the hidden option is removed and the env var is read directly in code (or handled via a consistent non-hidden option).

**Action:** Remove the two hidden deprecated `--nexus-key` options from `download` and `repair`, and their corresponding `nexus_key` parameters. Use `os.environ.get('NEXUS_API_KEY')` inline or add the standard `-k/--nexus-key` option matching `install`/`reinstall`.

## Finding 4: Redundant `IOError` alias in `cdn.py` (line 110)

**File:** `src/wabbajack/downloaders/cdn.py`, line 110

**What:**
```python
except (OSError, IOError) as e:
```
In Python 3, `IOError` is an alias for `OSError` — they are literally the same class. Catching both is identical to catching `OSError` once.

The same pattern is not present elsewhere in the codebase (all other files use `OSError` alone). This is the sole instance.

**Why safe:** Pure cleanup — `except OSError` catches all the same exceptions as `except (OSError, IOError)`. Zero behavioral change.

**Action:** Replace `except (OSError, IOError) as e:` with `except OSError as e:` on line 110.

## Finding 5: `InstallState.save()` is a trivial public wrapper around `_save()` with no callers (state.py lines 75-76)

**File:** `src/wabbajack/state.py`, lines 75–76

**What:**
```python
def save(self):
    self._save()
```
This is a one-line public delegation to the private `_save()`. A project-wide grep for `state.save()` or `.save()` in installer.py yields zero results — this public method is never called. All saving happens through the property setters (`phase`, `mark_hash_done`, `mark_complete`, `reset`) which call `_save()` directly.

**Why safe:** No call sites exist outside `state.py` itself. The method is not part of any interface contract. If future code needs to force a flush, it can call `_save()` directly or the method can be re-added at that point. Removing it simplifies the public API surface and eliminates confusion about when to call `save()` vs trusting the auto-save behavior of the setters.

**Action:** Remove `save(self)` (lines 75–76) from `InstallState`. Net reduction: 2 lines.
